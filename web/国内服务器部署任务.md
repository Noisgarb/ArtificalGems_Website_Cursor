# 国内服务器部署任务清单

部署到国内云服务器后，国内访客可直接访问，无需 VPN。按下面顺序执行即可。

---

## 一、准备阶段

| 序号 | 任务 | 说明 |
|------|------|------|
| 1 | **购买国内云服务器（VPS）** | 任选：阿里云 ECS、腾讯云 CVM、华为云 ECS 等，地域选「中国」、系统选 **Ubuntu 22.04** 或 **CentOS 7/8**。配置建议：1 核 2G 起，带宽按需。 |
| 2 | **开放安全组/防火墙端口** | 在云控制台放行：**22**（SSH）、**80**（HTTP）、**443**（HTTPS）。 |
| 3 | **（可选）准备域名与备案** | 若用域名访问且服务器在**中国大陆**，需做 **ICP 备案**（约 1–2 周）。备案通过后再解析域名；未备案前可先用 **服务器公网 IP** 访问。 |

---

## 二、服务器环境

| 序号 | 任务 | 说明 |
|------|------|------|
| 4 | **SSH 登录服务器** | `ssh root@你的公网IP`（或使用云控制台的密钥/密码）。 |
| 5 | **安装 Node.js 18+** | 例如 Ubuntu：`curl -fsSL https://deb.nodesource.com/setup_18.x \| sudo -E bash - && sudo apt install -y nodejs`。验证：`node -v`、`npm -v`。 |
| 6 | **安装 Nginx** | `sudo apt install -y nginx`（Ubuntu）或 `sudo yum install -y nginx`（CentOS）。用于反向代理和 HTTPS。 |
| 7 | **（可选）安装 Git** | 若未预装：`sudo apt install -y git` 或 `sudo yum install -y git`。 |

---

## 三、部署应用

| 序号 | 任务 | 说明 |
|------|------|------|
| 8 | **克隆代码** | `cd /var/www`（或自定目录），`sudo git clone https://github.com/你的用户名/ArtificalGems_Website_Cursor.git`，再 `cd ArtificalGems_Website_Cursor/web`。 |
| 9 | **安装依赖** | `npm install`（国内慢可设镜像：`npm config set registry https://registry.npmmirror.com`）。 |
| 10 | **配置环境变量** | `cp .env.example .env`，编辑 `.env`：<br>• **DATABASE_URL**：国内单机用 SQLite 填 `file:./data/prod.db`；若用云数据库则填 PostgreSQL 连接串。<br>• **JWT_SECRET**：`openssl rand -base64 32` 生成一串，填进去。 |
| 11 | **数据库：SQLite 时** | `mkdir -p data`，`npx prisma generate`，`npx prisma db push`，`npm run db:seed`。若 schema 当前是 PostgreSQL，需先改回 `provider = "sqlite"` 再执行（见下方「Prisma 用 SQLite」）。 |
| 12 | **数据库：PostgreSQL 时** | 使用云数据库（如阿里云 RDS）时，保持 `provider = "postgresql"`，在 `.env` 填好 `DATABASE_URL` 后执行 `npx prisma db push`、`npm run db:seed`。 |
| 13 | **构建与试运行** | `npm run build`，`npm run start`。本机 `curl http://127.0.0.1:3000` 能访问即成功。Ctrl+C 停掉后进入下一步。 |

---

## 四、常驻与对外访问

| 序号 | 任务 | 说明 |
|------|------|------|
| 14 | **用 PM2 常驻进程** | `sudo npm install -g pm2`，`pm2 start npm --name "gem-site" -- run start`，`pm2 save`，`pm2 startup`（按提示执行输出命令）。重启服务器后应用会自动启动。 |
| 15 | **配置 Nginx 反向代理** | 新建配置如 `sudo nano /etc/nginx/sites-available/gem-site`（Ubuntu），内容见下方「Nginx 配置示例」。再 `sudo ln -s /etc/nginx/sites-available/gem-site /etc/nginx/sites-enabled/`，`sudo nginx -t && sudo systemctl reload nginx`。 |
| 16 | **（推荐）配置 HTTPS** | 若已备案域名并解析到该服务器：`sudo apt install -y certbot python3-certbot-nginx`，`sudo certbot --nginx -d 你的域名`。按提示完成即可。 |

---

## 五、Prisma 使用 SQLite（国内单机）

若你在 Vercel 上已改为 PostgreSQL，国内单机想用 SQLite，需改回：

1. 编辑 `web/prisma/schema.prisma`，将 `provider = "postgresql"` 改回 `provider = "sqlite"`。
2. `.env` 中 `DATABASE_URL="file:./data/prod.db"`。
3. 在服务器 `web` 目录执行：`mkdir -p data`，`npx prisma generate`，`npx prisma db push`，`npm run db:seed`。

## 六、Nginx 配置示例

将 `你的域名` 换成实际域名或删掉 `server_name` 行、用 `default_server` 表示用 IP 访问：

```nginx
server {
    listen 80;
    server_name 你的域名;   # 或直接写 _;
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 七、部署后检查

| 项目 | 操作 |
|------|------|
| 首页 | 浏览器访问 `http://你的公网IP` 或 `http://你的域名`，能打开。 |
| 产品 / 联系 | 点击产品列表、联系我们，正常。 |
| 后台 | 访问 `/admin/login`，用 `admin` / `admin123` 登录，**登录后立即修改密码**。 |
| 本地上传 | 在后台新增产品时「本地上传」图片会存到服务器 `web/public/uploads`，可持久使用。 |

## 八、简要顺序汇总

1. 买国内 VPS → 放行 22/80/443  
2. SSH 装 Node、Nginx、Git  
3. 克隆仓库 → `web` 下 `npm install` → 配 `.env`（DATABASE_URL、JWT_SECRET）  
4. 按需选 SQLite 或 PostgreSQL → `prisma generate`、`db push`、`db:seed`  
5. `npm run build`、`npm run start` 试跑 → 用 PM2 常驻  
6. Nginx 反代 3000 → 可选 Certbot 配 HTTPS  
7. 用 IP 或域名访问并改后台密码  

更细的 Nginx/PM2 命令可参考 [公网部署.md](./公网部署.md) 中「方案二：部署到 VPS / 云服务器」。
