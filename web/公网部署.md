# 公网部署指南

本文说明如何将本站部署到公网，供任何人通过域名或 IP 访问。提供两种常见方案：**Vercel**（省心、适合前端为主）和 **VPS/云服务器**（可控、适合需要本地上传与 SQLite 的场景）。

**仓库说明**：GitHub 仓库根目录为 `ArtificalGems_Website_Cursor`，Next 应用在 **web/** 下。克隆后涉及构建、迁移等命令均在 `web` 目录执行；Vercel 的 Root Directory 填 `web`。

---

## 一、部署前准备

### 1. 环境变量

生产环境必须配置：

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `DATABASE_URL` | 数据库连接串 | 见下方「数据库」 |
| `JWT_SECRET` | 后台登录 Token 签名密钥，务必使用随机长字符串 | 可用 `openssl rand -base64 32` 生成 |

### 2. 数据库选择

- **开发/自建单机**：可继续用 SQLite，`DATABASE_URL="file:./dev.db"`。
- **Vercel / 多实例 / 云函数**：必须用 **PostgreSQL**（或 MySQL），因为 SQLite 无法在无状态环境持久化。

若改用 PostgreSQL：

1. 在 Prisma 中把数据库改为 PostgreSQL（见下方「Prisma 改用 PostgreSQL」）。
2. 在云服务商创建数据库（如 [Neon](https://neon.tech)、[Supabase](https://supabase.com)、[Vercel Postgres](https://vercel.com/storage/postgres)、阿里云 RDS 等），拿到连接串。
3. 设置 `DATABASE_URL`。Neon 连接串格式示例：`postgresql://用户:密码@ep-xxx-pooler.region.aws.neon.tech/库名?sslmode=require&channel_binding=require`（以 Neon 控制台显示为准）。

### 3. Prisma 改用 PostgreSQL（仅当使用 PostgreSQL 时）

编辑 `prisma/schema.prisma`，将：

```prisma
datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}
```

改为：

```prisma
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

然后在 **web** 目录下执行迁移与种子前，需先设置 `DATABASE_URL`（见下方「本地设置 DATABASE_URL 指令」），再执行：

```bash
cd web
npx prisma generate
npx prisma db push
npm run db:seed
```

**本地设置 DATABASE_URL 指令（与 Vercel 中填写的连接串一致）**

Neon 连接串通常含 `?sslmode=require&channel_binding=require`，内含 `&` 等特殊字符，需按下面方式设置，避免被终端解析错误。

- **Windows PowerShell**（推荐用单引号包裹整串）：
  ```powershell
  cd ArtificalGems_Website_Cursor\web
  $env:DATABASE_URL='postgresql://用户:密码@ep-xxx-pooler.region.aws.neon.tech/库名?sslmode=require&channel_binding=require'
  npx prisma generate
  npx prisma db push
  npm run db:seed
  ```

- **Windows CMD**（用双引号包裹整串）：
  ```cmd
  cd ArtificalGems_Website_Cursor\web
  set "DATABASE_URL=postgresql://用户:密码@ep-xxx-pooler.region.aws.neon.tech/库名?sslmode=require&channel_binding=require"
  npx prisma generate
  npx prisma db push
  npm run db:seed
  ```

- **Linux / macOS**：
  ```bash
  cd ArtificalGems_Website_Cursor/web
  export DATABASE_URL='postgresql://用户:密码@ep-xxx-pooler.region.aws.neon.tech/库名?sslmode=require&channel_binding=require'
  npx prisma generate
  npx prisma db push
  npm run db:seed
  ```

请将上述中的 `postgresql://用户:密码@...` 替换为你在 Neon 控制台复制、并在 Vercel 中填写的完整连接串。

## 二、方案一：部署到 Vercel（推荐快速上线）

- **优点**：免费额度、自动 HTTPS、与 Next.js 兼容好、连 Git 自动部署。
- **注意**：运行环境为无状态，**不能**使用 SQLite，**不能**把上传文件写到服务器本地；需使用云数据库 + 云存储（或暂时只用图片 URL，不上传本地文件）。

### 步骤概要

1. **代码推送到 Git**  
   将项目推到 GitHub / GitLab / Bitbucket，确保 Vercel 能访问该仓库。

2. **在 Vercel 创建项目**  
   - 打开 [vercel.com](https://vercel.com) 并登录。  
   - 点击 **Add New → Project**，导入你的 GitHub 仓库（如 `ArtificalGems_Website_Cursor`）。  
   - **Root Directory** 填：`web`（本仓库 Next 应用在根目录下的 `web` 文件夹中）。  
   - 构建命令：`npm run build`（或 `pnpm build`）。  
   - 输出目录保持默认，无需改。

3. **配置环境变量**  
   在 Vercel 项目 **Settings → Environment Variables** 中增加：
   - `DATABASE_URL`：PostgreSQL 连接串（必填）。  
   - `JWT_SECRET`：生产用随机密钥（必填）。

4. **数据库**  
   - 在 [Neon](https://neon.tech) 或 [Supabase](https://supabase.com) 等创建免费 PostgreSQL，复制连接串到 `DATABASE_URL`。  
   - 或使用 Vercel 的 **Storage → Postgres** 创建数据库并自动注入 `DATABASE_URL`。

5. **首次部署后执行迁移与种子**  
   - Vercel 部署不会自动执行 `prisma db push` / `prisma migrate`。  
   - 在**本地**进入 `web` 目录，先设置环境变量再执行迁移与种子（见下方「本地设置 DATABASE_URL 指令」）。

6. **图片上传说明（Vercel）**  
   当前「本地上传」是写入 `public/uploads`，在 Vercel 上**无法持久化**。可选做法：  
   - 暂时只使用「输入图片 URL」，不开放本地上传；或  
   - 后续改为接入 Vercel Blob / 阿里云 OSS / AWS S3 等对象存储，上传接口改为写这些服务。

7. **部署与访问**  
   保存环境变量后重新部署，Vercel 会给出 `https://xxx.vercel.app`，即公网访问地址。可绑定自己的域名（在 Vercel 项目 Settings → Domains）。

## 三、方案二：部署到 VPS / 云服务器（推荐要本地上传时）

- **优点**：可继续用 SQLite、本地上传存到 `public/uploads`、完全可控。  
- **适用**：有一台带公网 IP 的 Linux 服务器（阿里云、腾讯云、华为云、Vultr 等）。

### 步骤概要

1. **服务器环境**  
   安装 Node.js 18+、npm（或 pnpm）。若用 Nginx 做反代，再装 Nginx。

2. **上传代码并安装依赖**  
   ```bash
   cd /var/www   # 或你希望的目录
   git clone https://github.com/你的用户名/ArtificalGems_Website_Cursor.git
   cd ArtificalGems_Website_Cursor/web
   npm install
   ```

3. **配置环境变量**  
   ```bash
   cp .env.example .env
   # 编辑 .env，设置：
   # DATABASE_URL="file:./prod.db"   # 若用 SQLite（单机推荐）
   # 或 DATABASE_URL="postgresql://..."  # 若用 PostgreSQL
   # JWT_SECRET="用 openssl rand -base64 32 生成的长随机串"
   ```

4. **数据库（SQLite 示例）**  
   若用 SQLite，确保路径可写，例如：
   ```bash
   DATABASE_URL="file:./data/prod.db"
   mkdir -p data
   npx prisma db push
   npm run db:seed
   ```

5. **构建与启动**  
   ```bash
   npm run build
   npm run start
   ```
   默认监听 3000 端口。若只在本机访问，外网需通过 Nginx 反向代理。

6. **使用 Nginx 做反向代理（可选但推荐）**  
   这样可以用 80/443 端口、绑域名、配 HTTPS。示例（替换 `你的域名` 和 `服务器IP`）：

   ```nginx
   server {
       listen 80;
       server_name 你的域名;   # 或 服务器公网IP
       location / {
           proxy_pass http://127.0.0.1:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```

   重载 Nginx：`sudo nginx -t && sudo systemctl reload nginx`。

7. **HTTPS（推荐）**  
   在服务器上安装 **Certbot**，执行：
   ```bash
   sudo certbot --nginx -d 你的域名
   ```
   按提示即可得到免费证书并自动配置 443。

8. **进程常驻（推荐用 PM2）**  
   ```bash
   npm install -g pm2
   cd /var/www/ArtificalGems_Website_Cursor/web   # 与上面 clone 的路径一致
   pm2 start npm --name "gem-site" -- run start
   pm2 save
   pm2 startup
   ```
   这样重启服务器后 Next 也会自动启动。

9. **访问**  
   浏览器打开 `http://你的域名` 或 `http://服务器公网IP`；配置 HTTPS 后使用 `https://你的域名`。

## 四、Vercel 部署成功后的必做任务

构建通过后，要让线上站真正可用，请按顺序完成：

| 序号 | 任务 | 说明 |
|------|------|------|
| 1 | **在 Vercel 配置环境变量** | Settings → Environment Variables：添加 `DATABASE_URL`（PostgreSQL 连接串）、`JWT_SECRET`（随机长串）。保存后需重新部署一次。 |
| 2 | **创建 PostgreSQL 数据库** | 若还没有：在 [Neon](https://neon.tech) 或 [Supabase](https://supabase.com) 创建免费库，复制连接串填入 Vercel 的 `DATABASE_URL`。 |
| 3 | **Prisma 改为 PostgreSQL** | 编辑 `web/prisma/schema.prisma`，将 `provider = "sqlite"` 改为 `provider = "postgresql"`，保存。 |
| 4 | **本地执行迁移与种子** | 在 `web` 目录下按「本地设置 DATABASE_URL 指令」设置与 Vercel 一致的连接串（PowerShell 用单引号、CMD 用 `set "DATABASE_URL=..."`），再执行 `npx prisma db push`、`npm run db:seed`。完成后提交并推送代码，触发 Vercel 重新部署。 |
| 5 | **验证线上站** | 访问首页、/products、/contact；打开 /admin/login 用种子账号登录（如 `admin` / `admin123`），**登录后立即在后台修改密码**。 |

说明：Vercel 为无状态环境，**不能使用 SQLite**，必须用 PostgreSQL；本地上传图片在 Vercel 上不会持久化，可暂时只用「图片 URL」录入。

## 五、部署后检查（通用）

- 前台：打开首页 → 产品列表 → 产品详情 → 联系我们，能正常浏览和提交询盘。  
- 后台：访问 `https://你的域名/admin/login`，用种子账号（如 `admin` / `admin123`）登录，**首次上线后请立即修改密码**。  
- 若使用 SQLite（仅 VPS），确认 `DATABASE_URL` 指向的路径在服务器上可写；若使用 PostgreSQL，确认能从部署环境连上数据库（防火墙、白名单、SSL 参数等）。

## 六、简要对比

| 项目 | Vercel | VPS/云服务器 |
|------|--------|----------------|
| 成本 | 免费额度可用 | 需购买服务器 |
| 数据库 | 必须 PostgreSQL（或兼容） | SQLite / PostgreSQL 均可 |
| 本地上传图片 | 不支持，需对象存储 | 支持，存 `public/uploads` |
| 域名 / HTTPS | 内置 | 需自配 Nginx + Certbot 等 |
| 适合 | 快速上线、无运维 | 需要上传与完全可控 |

按你的需求（要本地上传且想省心 → 可先 Vercel + 仅 URL 传图；要本地上传且长期用 → 选 VPS）选择其一即可。
