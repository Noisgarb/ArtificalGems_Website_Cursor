# 网站部署与让他人访问说明

本文中的命令均在仓库的 **web** 目录下执行（即 `ArtificalGems_Website_Cursor/web`）。

## 当前网站能力

- **前台（访客可浏览）**：首页、产品列表、产品详情、联系我们/询盘表单。无需登录即可访问。
- **后台（管理员）**：`/admin/login` 登录后管理产品、询盘、配置等。

## 让他人能够访问的几种方式

### 1. 本机开发时，让同一局域网的人访问

默认 `npm run dev` 只监听 localhost，他人无法访问。

- 使用 **局域网访问** 脚本：
  ```bash
  npm run dev:lan
  ```
  启动后，在同一 WiFi/局域网内的设备可通过你的电脑 IP 访问，例如：`http://192.168.1.100:3000`（将 IP 换成你本机在局域网中的实际 IP）。

### 2. 生产环境运行（本机或服务器）

```bash
npm run build
npm run start
```

- 若希望在同一局域网被访问，需让 Next 监听所有网卡。可设置环境变量后再执行 `npm run start`：
  - Windows (PowerShell)：`$env:HOSTNAME="0.0.0.0"; npm run start`
  - Linux/macOS：`HOSTNAME=0.0.0.0 npm run start`
- 此时他人可通过 `http://你的IP:3000` 访问。

### 3. 部署到公网（推荐正式对外时使用）

将项目部署到公网服务器或云平台后，任何人通过域名或公网 IP 即可访问。

**详细步骤请查看：[公网部署.md](./公网部署.md)**，其中包含：

- **Vercel**：与 Next.js 兼容，连 Git 自动部署；需使用 PostgreSQL 与云存储（或暂只用图片 URL）。
- **VPS/云服务器**：可用 SQLite 或 PostgreSQL，本地上传图片可持久化，用 Nginx + PM2 配置域名与 HTTPS。

## 数据库与上传文件说明

- **SQLite**：当前默认使用本地 SQLite 文件。部署到多实例或云函数时，建议改为 PostgreSQL/MySQL 等可集中访问的数据库，并配置 `DATABASE_URL`。
- **本地上传图片**：存放在 `public/uploads`。若部署到无状态环境（如部分云函数），上传文件可能无法持久化，届时需改为对象存储（如 OSS、S3）。

## 总结

| 场景           | 做法 |
|----------------|------|
| 开发时给同事/同 WiFi 的人看 | `npm run dev:lan`，对方访问 `http://你电脑IP:3000` |
| 本机或内网服务器长期运行     | `npm run build` + `npm run start`，需要外网访问时设置 `HOSTNAME=0.0.0.0` |
| 正式对外、任何人可访问     | 部署到 Vercel/自建服务器等公网环境，并配置域名与数据库 |
